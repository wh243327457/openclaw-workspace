# 子代理系统方案

## 概述

主代理（你）可以创建和管理多个子代理。每个子代理：
- 独立的 OpenClaw 实例 + 独立 workspace
- 绑定一个人的微信，该用户可通过微信配置
- 继承主代理的基础技能，无系统权限
- 独立的记忆系统，主代理可读，子代理不可读主代理的

## 架构

```
主代理 (main)
├── workspace/          ← 你的完整工作空间
├── 可见: 所有子代理的聊天记录和状态
├── 可操作: 创建/停止/配置子代理
│
├── 子代理 A (agent-alice)
│   ├── workspace-alice/  ← 独立工作空间
│   ├── 微信绑定: Alice 的微信
│   ├── 技能: 从主代理继承（过滤后）
│   ├── 记忆: 独立，主代理可读
│   └── 无系统权限: 不能改主配置、不能看主记忆
│
└── 子代理 B (agent-bob)
    ├── workspace-bob/
    ├── 微信绑定: Bob 的微信
    └── ...
```

## 实现方案

### 方案 1：多 OpenClaw 实例（推荐）

每个子代理是一个独立的 OpenClaw 进程，通过 Docker 或 systemd 管理。

**优点**：完全隔离，安全，天然支持独立微信绑定
**缺点**：资源占用多，管理复杂

```
docker run openclaw/main ...     # 主代理，端口 18789
docker run openclaw/sub-alice ... # 子代理 A，端口 18790
docker run openclaw/sub-bob ...   # 子代理 B，端口 18791
```

### 方案 2：单实例多 Agent（轻量）

在同一个 OpenClaw 实例中，通过 agent-team 配置多个 agent，每个 agent 有自己的：
- workspace 子目录
- 微信绑定
- 技能过滤规则
- 记忆隔离

**优点**：资源省，管理简单
**缺点**：隔离不够彻底，安全风险

## 技能继承规则

```
主代理技能                    子代理可继承
─────────────                ──────────
memory-keeper         →      ✅ 只读版（不能修改主记忆）
self-growth           →      ✅ 
memory-system         →      ❌ 系统权限
memory-sync-policy    →      ❌ 系统权限
memory-bootstrap      →      ❌ 系统权限
agent-team-system     →      ❌ 系统权限
healthcheck           →      ❌ 系统权限
```

子代理技能 = 公开仓库的模板技能 + 主代理指定的可继承技能

```
├── MEMORY.md                  ├── MEMORY.md（独立）
├── memory/                    ├── memory/（独立）
│   ├── 2026-03-29.md          │   ├── 2026-03-29.md
│   └── ...                    │   └── ...
│                              │
主代理可读子代理 ✅              子代理不可读主代理 ❌
（通过文件系统符号链接）          （工作空间隔离）
```

## 微信绑定

每个子代理绑定独立的微信：
- 主代理: 微信 A（你自己的）
- 子代理 Alice: 微信 B（为 Alice 创建的 bot 微信）
- 子代理 Bob: 微信 C

子代理的微信用户（如 Alice）可以：
- 对话和日常使用
- 发送配置指令调整子代理
- 查看子代理的状态

不能：
- 看到主代理的聊天
- 访问系统级配置
- 影响其他子代理

## 用户通过微信调整子代理

子代理识别配置指令：

```
Alice 发送: "你叫什么名字？"
→ 子代理正常对话

Alice 发送: "/设置 名字 小蓝"
→ 子代理更新 IDENTITY.md

Alice 发送: "/设置 性格 温柔一点"
→ 子代理更新 SOUL.md

Alice 发送: "/设置 记忆 开启自动总结"
→ 子代理更新配置

Alice 发送: "/状态"
→ 子代理返回当前配置和状态
```

## 主代理管理界面

在主代理的配置 UI 中新增「子代理管理」Tab：

- 列表：所有子代理名称、状态、绑定微信、最后活跃时间
- 创建：输入名称、微信 ID、选择继承的技能
- 配置：编辑子代理的 SOUL.md、USER.md、技能
- 监控：查看子代理的聊天记录和记忆
- 停止/启动：控制子代理运行状态

## 实现步骤

### 第一阶段：基础架构
1. 创建子代理工作空间模板
2. 实现子代理的技能过滤机制

### 第二阶段：微信绑定
4. 配置子代理的微信 channel
5. 实现配置指令识别和处理

### 第三阶段：管理能力
6. 主代理的子代理管理命令
7. 记忆隔离和读取权限
8. Web UI 管理界面

### 第四阶段：优化
9. 子代理健康监控
10. 自动故障恢复
11. 子代理间的通信（可选）
